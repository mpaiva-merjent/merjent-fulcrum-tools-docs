Report Template Cheat Sheet

Quick reference for building Merjent PDF report templates with ReportHelpers (M), MfldColl, Mfld, and SecOpt. M is already instantiated in merjent-tools-report.mmin.js — drop the dist file inside the region markers and use M directly.

//#region !START MERJENT FULCRUM TOOLS!
// ... paste merjent-tools-report.mmin.js content here ...
//#endregion !END MERJENT FULCRUM TOOLS!

// M, MfldColl, Mfld, SecOpt are available from here on

Quick Boilerplate

The most common section pattern — three lines of setup, one line to render.

<%
  const secDn  = 'site_information';
  const dns    = FIELDNAMES(secDn, {sections: false});
  const mFlds  = M.mkMflds(dns);
  const secOpt = new SecOpt(null, LABEL(secDn), null);
%>
<%- M.buildDefaultSection(secDn, mFlds, secOpts) %>

Golden Rules

  • Grid spans must total 12 per row — otherwise the grid wraps in unexpected ways.
  • Use M.fldVal() (not VALUE()) for rendering — handles nulls, choices, addresses, yes/no.
  • Prefer FIELDNAMES() over hardcoded data_name arrays — the report adapts when the form changes.
  • Preserve region markers — the Python deploy tool (merjent-fulcrum-py/src/tools/mtools/update_mtools.py) uses them to swap the dist block without touching your custom code.
  • Don't re-instantiate M — the dist file already does const M = new ReportHelpers().

M.buildSimpleCell(dn_or_mfld, i) Single horizontal cell

Photo Builders

Method Purpose

M.buildRepeatablePhotos(repDn, photoDn, secOpt, opts, firstpg, cb) Photos grouped by repeatable item
M.buildPhotoGrid1x3(photos, secOpt, opts, firstpg) 1×3 photo grid layout
M.buildPhotoWrapper(photoObj, opts) Single photo + caption block
M.buildCaptionRows(photoObj, photoLoc, opts) Customize caption row order/content
M.photoRequest(recIDs, appID, photoDn, orderDn, qFlds) Fetch photo records via SQL
M.photoHashRequest(recIDs, appID, photoDn) Fetch only photos with #-prefixed captions
M.photoRequestMeta(photo_id) Get lat/lon/date/direction from a photo
M.mkPhotoOarr(rows, dn, photoLbl) Build photo object array from SQL rows
M.photoCaptionNumber(photoOarr, delim, numFld) Split # prefix into its own field
M.photoRemTag(captionStr) Strip # or ! prefix from caption

Photo opts Options

Key Type Description
labelField string Caption label row source (default: 'item_no')
showLocation boolean Show lat/lon row (default: true)
showDate boolean Show EXIF date (default: false)
showDirection boolean Show compass direction (default: false)
fields array Extra caption rows: [{label, value}] or ['field_dn']

addXtra boolean Legacy: dump all extra photoObj keys

Table / Repeatable Data

Method Purpose

MfldColl & Mfld Cheat Sheet

MfldColl is the array of Mflds passed into section builders. Access individual fields with .m(dn).

const mFlds = M.mkMflds(dns);

mFlds.m('project_name').span = 12;     // full width
mFlds.m('habitat_types').opt = '-cb';  // render choices as checkboxes
mFlds.m('notes').opt = '-hn';          // hide if null

Mfld Properties You'll Touch

Property Type Purpose
span number Grid column span (1–12, default 4)
opt string Render option string — concatenated flags
data_name string Field data name
label string Field label (from schema)
_type string Fulcrum field type (read-only)
hidden boolean Hidden in grid
parent string Direct parent data name (repeatables)

.opt Flags

Chain multiple with no separator — e.g. '-cb-rdo'.

Flag Effect
-na Replace null values with N/A instead of ---
-hn Hide field entirely when value is null
-cb Render choice values as vertical checkboxes
-hcb Modifier for -cb: horizontal checkboxes (only shows checked)
-rdo Modifier for -cb: replace checkboxes with radio buttons
-syn Short yes/no: Y / N / N/A
-yon Yes-or-no only: hide the neutral label

SecOpt Cheat Sheet

new SecOpt(prefix, center, suffix, lblIndex);
Parameter Type Purpose
prefix string Text before the centered title
center string Main title text (overrides data_name / rep_data_name)
suffix string Text after the centered title
lblIndex boolean Append repeatable index number after prefix

SecOpt Properties

Property Type Default Purpose
hide boolean false Skip rendering entirely
data_name string '' Use LABEL(data_name) as the centered title
rep_data_name string '' Pull title from a repeatable field value (uses index)
index number null Repeatable index (set automatically by buildRepeatable*)
css string 'mjnt-sectionHeader spacer-top' CSS classes for the header

Common Patterns

// Basic titled section
new SecOpt(null, LABEL(secDn), null);

// Section with note/suffix
new SecOpt(null, 'Weather', 'recorded at survey start');

// Conditional hide
const opts = new SecOpt(null, 'Site Photos', null);
opts.hide = VALUE('site_photos').length === 0;

// Numbered subsection header inside a repeatable
new SecOpt('Observation', null, null, true);  // → "Observation 1", "Observation 2"...

  const secDn = 'equipment_checklist';

  const secOpt = new SecOpt(null, LABEL(secDn), null);

Repeatable as Subsections

<%
  const repDn = 'species_observations';
  const dns   = FIELDNAMES(repDn);
  const mFlds = M.mkMflds(dns);
  const secOpt = new SecOpt(null, LABEL(repDn), null);
%>
<div class="mjnt-sectionHeader spacer-top"><%= LABEL(repDn) %></div>
<%- M.buildRepeatableSubsec(repDn, mFlds, secOpt) %>

Repeatable as Open Table

<div class="clump">
<%
  const secDn  = 'survey_points';
  const dns    = FIELDNAMES(secDn);
  const repID  = '7d00-...-child-form-id';
  const rows   = M.repRequestRecords([RECORDID()], FORM().id, repID, null, M.addCH(dns));
  const divRows = M.rows2Divs(rows, dns);
%>
  <div class="mjnt-sectionHeader spacer-top"><%= LABEL(secDn) %></div>
  <div class="grid-lo open-table" style="grid-template-columns:repeat(5, 1fr)">
    <div class="header"><%- M.makeTableHeaders(dns) %></div>
    <div class="rows"><%- divRows %></div>
  </div>
</div>

Common Mistakes

Mistake Fix
<%= VALUE('address') %> shows [object Object] Use <%= M.fldVal('address') %>
Grid wraps unexpectedly Make sure span values in a row sum to 12
Re-instantiating M inside template Remove — dist file already did new ReportHelpers()
Editing ReportTemplateBlocks.js directly Edit templates in report-templates/html-blocks.json, then rebuild
Hardcoded field list breaks on schema edits Use FIELDNAMES(secDn, {sections: false})
Region markers missing Wrap dist in //#region !START MERJENT FULCRUM TOOLS! / //#endregion !END MERJENT FULCRUM TOOLS! so the Python deploy tool (update_mtools.py) can update it
