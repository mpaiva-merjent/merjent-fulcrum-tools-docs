Standalone HTTP request class for Fulcrum REST & Query APIs

Description

Wraps Fulcrum's REQUEST() function with consistent error handling, JSON parsing, and auth headers. Supports query, fetch, update, batch, and external API calls.

Pattern: Standalone class instantiated via factory M.mkRequest() or directly.

Parameters
Name Type Attributes Default Description
config Object <optional>
 {}

Optional initial configuration

Name Type Attributes Description
onError function <optional>

Custom error handler overriding the default
Examples

Via factory method

const req = M.mkRequest()
req.query("SELECT * FROM \"My App\"", function(rows) {
  SETVALUE('count', rows.length)
})

Fetch-modify-update

const req = M.mkRequest()
req.modifyRecord(recID, function(record) {
  record.record.form_values['a1b2'] = 'New Value'
  return record
}, function() {
  ALERT('Updated!')
})

Batch update

const req = M.mkRequest()
req.batchUpdate(recordIDs, function(record) {
  record.record.form_values['c3d4'] = 'Batch Value'
  return record
}, { completeMsg: 'All records updated!' })

External API

const req = M.mkRequest()
req.external({
  url: 'https://api.example.com/data',
  qs: { lat: LATITUDE(), lon: LONGITUDE() }
}, function(data) {
  SETVALUE('result', data.value)
})

modifyRecord

Fetch-modify-update a single record

Description

Fetches a record, passes it to buildFn for modification, then PUTs the result. If buildFn returns null/undefined, the update is skipped.

Parameters
Name Type Attributes Description
recID string

_record_id
buildFn function

Receives record object, must return modified record (or null to skip)
callback function <optional>

Called after successful PUT
Examples
req.modifyRecord(recID, function(record) {
  record.record.form_values['a1b2'] = 'Updated'
  return record
}, function() {
  ALERT('Done!')
})
